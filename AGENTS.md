# AGENTS.md — 에이전트 작성 규칙

이 레포에 이슈/코멘트를 작성하는 모든 AI 에이전트(Claude Code, Claude Desktop, 기타)는 아래 규칙을 따른다.
목적: 작성자가 누구든 **일관된 형식과 퀄리티**로 컨텍스트가 쌓이게 하는 것.

## 언어

- 모든 이슈와 코멘트는 **한국어**로 작성한다. (기술 용어, 고유명사는 원어 허용)

## 이슈 제목

- 형식: `[주제라벨] 핵심을 한 줄로`
- 예: `[business] 초기 타겟을 개인투자자 vs 소규모 자문사 중 어디로 할지`
- 예: `[decision] 리치 콘텐츠 보관소를 Notion으로 확정`

## 이슈 본문 — 필수 구조

과정 보존이 목적이다. 결론만 쓰지 말고 아래 5개 섹션을 채운다.
해당 없는 섹션은 `(해당 없음)`으로 남기되 삭제하지 않는다.

```markdown
## 배경 / 고민
무엇이 이 생각을 촉발했는가. 사용자가 표현한 고민과 질문을 원문에 가깝게 보존한다.

## 문제 정의
해결하려는 문제를 한두 문장으로.

## 검토한 선택지
각 선택지와 장단점, 기각한 것은 기각 이유까지.

## 결론 / 현재 상태
결론이 났으면 결론과 근거. 진행 중이면 현재까지의 판단과 다음 확인할 것.

## References
관련 이슈(#번호, 타 레포는 owner/repo#번호 풀 네임), Notion URL, 외부 링크.
```

## 작성 시 핵심 원칙

1. **사용자의 원문 고민을 보존한다.** 매끈하게 요약하면서 뉘앙스를 지우지 말 것. 사용자가 던진 질문 자체가 가치 있는 컨텍스트다.
2. **기각된 선택지와 기각 이유를 반드시 기록한다.** "왜 안 했는가"가 나중에 가장 값진 정보다.
3. **1이슈 1주제.** 두 가지 고민이 섞이면 이슈를 분리한다.
4. **후속 생각은 코멘트로.** 같은 주제의 새 생각은 새 이슈가 아니라 기존 이슈에 코멘트로 추가한다. 기존 이슈는 검색으로 먼저 확인할 것.
5. **출처를 구분한다 (가능한 상황에서).** 사용자가 직접 묻거나 이해한 내용과 에이전트가 제시한 내용을 구분해 기록한다. `배경 / 고민`에는 사용자의 질문·표현을 인용 위주로 담고, `검토한 선택지`의 각 항목에는 `(사용자 제안)` `(에이전트 제안)` `(공동 도출)` 태그를 붙인다. PRD/제품 방향처럼 여러 주체의 의견이 오가는 논의에서는 `사용자 원문 의도` / `AI 에이전트 피드백` / `사용자 최종 판단` / `반영 결과`로 더 세분화해도 된다. 구분이 모호하면 무리하게 나누지 않는다.

## 라벨 규칙

- **상태 라벨 정확히 1개**: `backlog` 또는 `debating` (open 이슈), close 시에도 유지
- **주제 라벨 1개 이상**: `idea` `business` `research` `feedback` `decision`
- **정의된 라벨 외 임의 생성 금지.** 새 라벨이 필요하면 사용자에게 제안만 하고, 승인 후 README 갱신과 함께 추가한다.

## 상태 전이 규칙

- 초기 아이디어/고민 → `backlog`로 open
- 검토가 시작되면 → `backlog` 제거, `debating` 부착
- 결론 도출 → 결론을 코멘트로 남기고 `decision` 라벨 추가 후 **close**
- 기각 → 기각 이유를 마지막 코멘트로 남기고 **close**
- 진행이 없는 기록형(이미 난 결정, 받은 피드백, 리서치 메모) → 생성 즉시 **close**
- closed는 폐기가 아니라 확정된 기록이다. 임의로 재오픈하지 말고, 재논의가 필요하면 새 이슈를 만들어 기존 이슈를 References로 연결한다.

## 금지 사항

- 사용자 승인 없이 이슈 삭제/제목 대량 변경/라벨 체계 변경 금지
- 여러 고민을 하나의 거대 이슈로 뭉치기 금지
- 영어로 작성 금지 (사용자가 요청한 경우 제외)
